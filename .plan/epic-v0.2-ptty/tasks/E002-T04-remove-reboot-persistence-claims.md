# E002-T04: Remove reboot persistence claims

## Objective

Ensure every public and release-facing text uses the correct persistence claim:
in-memory sessions survive SSH/client disconnects, not server reboot.

## Acceptance Criteria

- [x] `install.sh` does not say sessions survive server reboot.
- [x] Release workflow notes do not claim session state survives reboot.
- [x] README distinguishes daemon autostart from session persistence.
- [x] Claude guidance keeps the same invariant and contains no contradictory older copy.

## Tests

- `rg -n "survive.*reboot|survives.*reboot|crash-resistant|server reboot" README.md CLAUDE.md install.sh .github`
- Manual review of release notes generated by `.github/workflows/release.yml`.

## Notes

Allowed phrasing: systemd user service can recreate empty sessions after boot.
Forbidden phrasing: in-memory AI session context survives server reboot.

Implementation uses "empty sessions auto-create after boot" and
"disconnect-resistant" language instead of crash/server-reboot persistence.
